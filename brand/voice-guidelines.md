# Insight Series トーン＆マナー ガイドライン

## ブランドボイス

### コンセプト: Professional Intelligence

Insight Seriesは「プロフェッショナルに寄り添う、信頼できるパートナー」です。
高級コンサルティングファームのような洗練された印象を与えつつ、親しみやすさを持ちます。

### ブランドパーソナリティ

| 特性 | 説明 | 例 |
|------|------|-----|
| **知的** | データと根拠に基づいた提案 | 「分析結果から〜が最適です」 |
| **信頼** | 確実で安心感のある表現 | 「確実に保存されました」 |
| **効率的** | 無駄のない簡潔な表現 | 「完了」「3件処理済み」 |
| **先進的** | AI/技術の価値を伝える | 「AIが最適化しました」 |
| **誠実** | 正直で透明性のある表現 | 「処理に時間がかかります」 |

---

## 文体ルール

### 基本原則

```
✓ 敬体（です・ます調）を基本とする
✓ 簡潔で無駄のない表現
✓ 主語を省略せず明確に
✓ 専門用語は必要最小限に
✗ カジュアルすぎる表現を避ける
✗ 過度に堅い表現を避ける
✗ ！マークの多用を避ける
```

### トーンの段階

| レベル | 使用場面 | 例 |
|--------|---------|-----|
| **フォーマル** | 法務文書、利用規約 | 「本サービスを利用するものとします」 |
| **スタンダード** | UI全般、ヘルプ | 「設定を保存しました」 |
| **フレンドリー** | オンボーディング、成功メッセージ | 「準備が整いました」 |

## UIテキスト

### ボタンラベル

```
良い例:
  保存    編集    削除    キャンセル
  分析を開始    レポート作成    エクスポート

悪い例:
  保存する    編集します    削除！    やめる
  分析をスタートさせる    レポートを作る
```

**ルール:**
- 動詞は終止形（〜する → する ではなく「保存」「開始」）
- 2〜4文字を目安に簡潔に
- アイコンと併用する場合はラベルを省略可

### メニュー/ナビゲーション

```
良い例:
  ホーム    ダッシュボード    設定    ヘルプ
  ファイル    編集    表示    ツール

悪い例:
  ホーム画面へ    設定画面    ヘルプを見る
```

### 入力フィールド

```
ラベル: メールアドレス
プレースホルダー: example@company.com
ヘルプテキスト: 確認メールを送信します

ラベル: パスワード
プレースホルダー: 8文字以上
ヘルプテキスト: 大文字・小文字・数字を含めてください
```

### エラーメッセージ

```
良い例:
  「メールアドレスの形式が正しくありません」
  「ファイルサイズが上限（10MB）を超えています」
  「ネットワーク接続を確認してください」

悪い例:
  「エラーが発生しました」（具体性がない）
  「メールアドレスが間違っています！」（！の使用）
  「Error: Invalid email format」（英語のまま）
```

**エラーメッセージの構成:**
1. 何が問題か（具体的に）
2. どうすれば解決できるか（可能な場合）

### 成功メッセージ

```
良い例:
  「保存しました」
  「3件のファイルをエクスポートしました」
  「設定を更新しました」

悪い例:
  「保存成功！」
  「うまくいきました！」
  「Success!」
```

### 確認ダイアログ

```
タイトル: ファイルを削除しますか？
本文: この操作は取り消せません。選択した3件のファイルが完全に削除されます。
ボタン: [キャンセル] [削除]

タイトル: 変更を保存しますか？
本文: 保存せずに閉じると、変更内容が失われます。
ボタン: [保存しない] [キャンセル] [保存]
```

### 空状態（Empty State）

```
良い例:
  タイトル: まだデータがありません
  説明: 分析を開始すると、ここに結果が表示されます
  アクション: [分析を開始]

悪い例:
  「データがありません」（だけ）
  「0件」（だけ）
```

### ローディング状態

```
良い例:
  「分析中...」
  「データを読み込んでいます」
  「少々お待ちください」

悪い例:
  「Loading...」
  「ちょっと待ってね」
```

## 敬称・呼称

### ユーザーへの呼称

```
使用しない: お客様、ユーザー様、あなた様
推奨: 呼称を使わない（主語省略）

良い例:
  「プロフィールを更新しました」（主語省略）

悪い例:
  「お客様のプロフィールを更新しました」
```

### AI/システムの呼称

```
推奨: 主語を省略、または「Insight」
例:
  「分析が完了しました」
  「Insightが最適な提案を作成しました」

使用しない:
  「私が〜しました」
  「AIが〜」（単体では使用しない）
```

## 多言語対応

### 日本語優先

- 日本語UIを基本とし、英語は補助的に使用
- 技術用語は適度にカタカナ化

```
良い例: ダッシュボード、エクスポート、フィルター
悪い例: Dashboard、Export、Filter（英語のまま）
```

### 一般的な対訳

| 英語 | 日本語 |
|------|--------|
| Dashboard | ダッシュボード |
| Settings | 設定 |
| Export | エクスポート |
| Import | インポート |
| Filter | フィルター |
| Search | 検索 |
| Notification | 通知 |
| Profile | プロフィール |
| License | ライセンス |
| Upgrade | アップグレード |

## 禁止表現

### 使用禁止

```
✗ 「〜しちゃう」「〜だね」（カジュアルすぎ）
✗ 「！！」「？？」（記号の重複）
✗ 絵文字・顔文字（UIテキスト内）
✗ 「エラーです」（具体性なし）
✗ 「失敗しました」（ネガティブすぎ）
✗ 「お待ちください...」（...の使用）
```

### 言い換え例

| 避ける表現 | 推奨表現 |
|-----------|---------|
| エラーです | 〜を確認してください |
| 失敗しました | 〜できませんでした |
| 待ってください | 処理中です |
| ダメです | 〜が必要です |
| 押してください | 選択してください |
