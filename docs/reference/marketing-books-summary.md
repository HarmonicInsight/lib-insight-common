## ① 『新訳 ハイパワー・マーケティング』（ジェイ・エイブラハム 著）

### ■ 一言で言うと
**売上を伸ばす道は3つしかない。** その3つに対して「具体的に何をやるか」を網羅した、売り方の全体地図。

### ■ 核心フレームワーク ── 売上の3レバー

```
売上 ＝ 顧客数 × 客単価 × 購買頻度
```

| レバー | やること（具体施策） |
|--------|---------------------|
| **顧客数を増やす** | ・紹介プログラムの設計（既存顧客にインセンティブを与え紹介を仕組み化）<br>・ジョイントベンチャー（他社の顧客基盤を借りて自社製品を売る）<br>・リスクリバーサル（返金保証・成果保証でお客様の購買リスクをゼロにする）<br>・USP（Unique Selling Proposition）の明確化と訴求 |
| **客単価を上げる** | ・アップセル（上位プランの提案）<br>・クロスセル（関連商品・サービスのセット提案）<br>・価格の再設定（価値に見合った価格への見直し） |
| **購買頻度を上げる** | ・定期接触（メール、ニュースレター等で忘れられない仕組み）<br>・ロイヤルティプログラム<br>・期限付きオファーで再購入を促進 |

### ■ 特に重要な概念

| 概念 | 内容 |
|------|------|
| **USP** | 「なぜ他社ではなくあなたから買うべきか」を一文で言えるようにする。すべてのマーケティングの出発点 |
| **リスクリバーサル** | 顧客の不安を売り手が引き受ける。返金保証、成果保証、お試し期間など。これだけで成約率が劇的に上がる |
| **ジョイントベンチャー** | 自社にない顧客基盤・信用・チャネルを持つパートナーと組み、互いの資産を活用して売る |
| **卓越の戦略** | 顧客を「取引相手」ではなく「守るべき存在」と捉え、長期的な信頼関係を築く |

### ■ HARMONIC insight への示唆

- **USP**: 「世界唯一のBizRobo→aKaBot移行自動化」「手動の10分の1のコスト」は既に強力なUSP
- **リスクリバーサル**: IMA無料診断 → パイロット → 本契約の段階設計はリスクリバーサルそのもの
- **ジョイントベンチャー**: FPTチャネルは典型的なJV。他のSIerやコンサルファームとのJV拡大も検討可能
- **購買頻度**: SaaSモデル（月額課金）は購買頻度の問題を構造的に解決している

## ② 『実施する順に解説！「マーケティング」実践講座』（弓削徹 著）

### ■ 一言で言うと
**マーケティング施策を「やる順番」でそのまま解説。** 「次に何をすればいいか」が迷わなくなる段取り本。

### ■ 実施順序 ── そのまま目次になっている流れ

```
Step 1  市場調査
   ↓
Step 2  ターゲット設定
   ↓
Step 3  ネーミング
   ↓
Step 4  価格設定
   ↓
Step 5  流通チャネル設計
   ↓
Step 6  キャッチコピー作成
   ↓
Step 7  ブランディング
```

### ■ 各ステップの要点

| Step | やること | ポイント |
|------|---------|---------|
| **1. 市場調査** | 競合・顧客ニーズ・市場規模を把握する | 「感覚」ではなくデータで判断する。SONYの事例など、調査が製品の方向性を決めた実例が豊富 |
| **2. ターゲット設定** | 「誰に売るか」を具体的に絞り込む | 万人向けは誰にも刺さらない。ペルソナを明確にし、そこに集中する |
| **3. ネーミング** | 製品名・サービス名を戦略的に決める | 名前は最初の広告。覚えやすく、価値が伝わり、検索されやすい名前を設計する |
| **4. 価格設定** | 値付けを戦略的に行う | コストベースではなく「顧客が感じる価値」ベースで決める。価格は品質のシグナル |
| **5. 流通チャネル** | どこで・どうやって届けるかを決める | 直販 / 代理店 / EC / 店舗など、ターゲットに合ったチャネルを選択・設計する |
| **6. キャッチコピー** | 心に残る言葉で価値を伝える | 機能ではなく「ベネフィット（顧客が得る結果）」を語る。サントリー等の事例 |
| **7. ブランディング** | 一貫したイメージを構築する | ロゴ・色・トーン・顧客体験のすべてを統一し、信頼と記憶に残るブランドを作る |

### ■ HARMONIC insight への示唆

- **Step 2（ターゲット）**: 4サイトをターゲット別に分けている現戦略は本書の教えに合致
- **Step 3（ネーミング）**: 「InsightCast」「InsightMigration」等、Insight統一ブランドは良い設計
- **Step 5（チャネル）**: FPT経由と広告直販の2チャネルは段取りとして明確
- **やり残しの確認**: 各Stepを順にチェックし、抜け漏れがないか棚卸しに使える

## ③ 『ファンダメンタルズ×テクニカル マーケティング』（木下勝寿 著）

### ■ 一言で言うと
**EC年商100億円超の社長が、Webで売るための手法を83項目で全公開。** 「誰に何を言うか」と「どこでどう見せるか」の両輪。

### ■ 核心フレームワーク ── 2つの軸

```
┌─────────────────────────────────────────────────┐
│          Webマーケティングの両輪                    │
│                                                   │
│  ファンダメンタルズ        テクニカル               │
│  （誰に何を言うか）        （どこでどう見せるか）    │
│                                                   │
│  ・ターゲット分析          ・Web広告運用            │
│  ・訴求メッセージ設計      ・LP（ランディングページ）│
│  ・広告文の書き方          ・ABテスト               │
│  ・顧客インサイトの発掘    ・データ分析と改善        │
│  ・コピーライティング      ・クリエイティブ最適化    │
└─────────────────────────────────────────────────┘
```

### ■ 主要な実践項目

| カテゴリ | 具体的手法 |
|---------|-----------|
| **広告文の書き方** | ・「自分ごと化」させる冒頭文（ターゲットが「私のことだ」と思う言葉から始める）<br>・機能ではなく「変化（Before→After）」を語る<br>・具体的な数字を入れる（「10分の1のコスト」など） |
| **LPの構成** | ・ファーストビューで離脱を防ぐ（3秒で価値が伝わるか）<br>・問題提起 → 解決策 → 証拠 → CTA の流れ<br>・お客様の声・実績を中盤に配置 |
| **Web広告運用** | ・少額で複数パターンをテストし、勝ちパターンに集中投下<br>・CPA（顧客獲得単価）を基準に判断<br>・クリエイティブの差し替えサイクルを仕組み化 |
| **データ改善** | ・数値で判断する文化を徹底（感覚に頼らない）<br>・ABテストを常時回す<br>・LTV（顧客生涯価値）で広告投資を判断 |

### ■ HARMONIC insight への示唆

- **InsightCast（広告直販SaaS）** のWeb集客に直接適用可能
- **LP構成**: 各製品のランディングページを「問題提起→解決策→証拠→CTA」で再点検
- **自己証明マーケティング**: InsightCastで自社の広告を制作して成果を見せる戦略は、本書の「証拠を見せる」原則と完全に一致
- **ABテスト**: 広告文・LP・価格プランのテストを小規模から始める仕組みを構築すべき

## ④ 『売り方の正解 ― 小さなお店が生き残る50のヒント』（竹内謙礼 著 / 2026年1月刊）

### ■ 一言で言うと
**人も金も時間もない状況で、明日からすぐ試せる集客・販促手法を50個に整理。** 小さな規模のリアルな話。

### ■ 50のヒントの主要カテゴリ

| カテゴリ | 代表的な手法 |
|---------|-------------|
| **Googleマップ活用** | ・Googleビジネスプロフィールの最適化<br>・口コミの獲得と返信の仕組み化<br>・写真・投稿の定期更新 |
| **看板・アナログ施策** | ・看板広告の作り方（通行量×視認性×メッセージ）<br>・チラシ・DM・ポスティングの効果的な設計<br>・手書きPOPの威力 |
| **QRコード活用** | ・オフラインからオンラインへの導線設計<br>・店頭→LINE登録→リピート促進の流れ |
| **SNSに頼らない集客** | ・口コミ・紹介の仕組み化<br>・地域メディア・フリーペーパーの活用<br>・イベント・体験会の開催 |
| **即効性のある販促** | ・期間限定・数量限定オファー<br>・セット販売・まとめ買い促進<br>・リピーター向け特典 |

### ■ 本書の特徴

- 600以上の小規模事業者への取材に基づくリアルな事例
- 大企業の成功事例ではなく、「小さな会社・お店」の地に足がついた話
- 「SNSをやらなければ」というプレッシャーからの解放 ── アナログ施策でも十分戦える
- 1つ1つが独立した「ヒント」なので、自社に合うものだけ選んですぐ実践可能

### ■ HARMONIC insight への示唆

- **直接的な適用度は低め**（BtoB SaaSのため、店舗向け施策は対象外）
- ただし以下の考え方は参考になる:
  - **限られたリソースでの優先順位付け**: 少人数チームでの施策選定の考え方
  - **口コミ・紹介の仕組み化**: BtoBでも導入企業からの紹介は最強のチャネル
  - **オフライン→オンライン導線**: 展示会・セミナー→Webサイト→問い合わせの流れに応用

## ⑤ 『影響力の武器 ― なぜ、人は動かされるのか』（ロバート・チャルディーニ 著）

### ■ 一言で言うと
**人が「買う」と決断する6つの心理原則。** すべての販促手法の「なぜ効くか」がわかる世界的名著。

### ■ 6つの影響力の原則

| 原則 | メカニズム | 販促での活用例 |
|------|-----------|---------------|
| **① 返報性** | 人は何かをもらうと、お返しをしたくなる | 無料サンプル、無料診断、有益な情報提供 → 購入・契約へ |
| **② コミットメントと一貫性** | 一度「はい」と言うと、その後も一貫した行動を取りたくなる | 小さなYes（無料登録、資料請求）から始め、段階的に大きなYes（契約）へ導く |
| **③ 社会的証明** | 他の人がやっていると、正しいことだと感じる | 「お客様の声」「導入実績○○社」「○○業界No.1」 |
| **④ 好意** | 好きな人・信頼できる人からの提案は受け入れやすい | 営業担当の人柄、ブランドへの親しみ、共通点の提示 |
| **⑤ 権威** | 専門家・権威者の言葉は信じやすい | 「○○大学教授推薦」「業界専門家が開発」「特許取得」 |
| **⑥ 希少性** | 手に入りにくいものほど価値が高く感じる | 「期間限定」「残りわずか」「先着○名」「今だけ特別価格」 |

### ■ 各原則の深掘り

```
返報性         →  先に価値を与える。無料で役に立つことをすれば、相手は
                  「何かお返ししなければ」と感じる

コミットメント  →  フット・イン・ザ・ドア技法。小さな依頼から始めて、
                  徐々に大きな依頼へ。自分の行動と矛盾したくない心理

社会的証明     →  不確実な状況ほど「他の人はどうしているか」を参考にする。
                  類似した他者の行動が最も影響力が強い

好意           →  外見的魅力、類似性、称賛、接触頻度、協力関係が好意を生む。
                  好意ある相手からの要請は断りにくい

権威           →  肩書き、服装、装飾品（専門性の外的シンボル）だけでも
                  従順さが増す。実質的な専門性があればさらに強力

希少性         →  「失う」恐怖は「得る」喜びの約2倍の心理的インパクト。
                  以前は手に入ったものが手に入らなくなる時、最も強く作用
```

### ■ HARMONIC insight への示唆

| 原則 | 具体的な適用 |
|------|-------------|
| **返報性** | IMA無料診断は返報性の完璧な活用。「無料でここまでやってくれた」→ 本契約へ |
| **コミットメント** | 無料 → パイロット → 本契約のステップ設計は、小さなYesの積み重ね |
| **社会的証明** | 導入実績・成功事例の公開。「○○社が導入」「移行成功率○○%」 |
| **権威** | FPT（大手IT企業）との提携は権威の証明。技術ブログ・登壇も権威構築に有効 |
| **希少性** | 「世界唯一のBizRobo→aKaBot移行対応」は希少性そのもの。もっと前面に打ち出すべき |
| **好意** | 定期的な接触（メルマガ、セミナー）で好意と信頼を構築 |

## ⑥ 『THE MODEL』（福田康隆 著）

### ■ 一言で言うと
**BtoB営業を4つの専門チームに分業し、連携させる仕組み。** 見込み客の獲得から受注・継続までの流れを科学的に設計する。

### ■ 核心フレームワーク ── 4つのプロセス

```
マーケティング → インサイドセールス → フィールドセールス → カスタマーサクセス
   (集客)         (見込み客育成)        (商談・受注)          (継続・拡大)
```

### ■ 各プロセスの役割と指標

| プロセス | 役割 | 主要KPI |
|---------|------|---------|
| **マーケティング** | 見込み客（リード）を獲得する | リード獲得数、チャネル別CPA |
| **インサイドセールス** | リードを精査し、商談化する | 商談化率（SQL転換率）、架電数、接触率 |
| **フィールドセールス** | 商談を進め、受注する | 受注率、案件単価、商談サイクル |
| **カスタマーサクセス** | 導入後の成功を支援し、継続・拡大を図る | 解約率（チャーン）、アップセル率、NPS |

### ■ 分業と連携の設計

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│マーケティング │    │インサイド     │    │フィールド     │    │カスタマー     │
│              │    │セールス       │    │セールス       │    │サクセス       │
│              │    │              │    │              │    │              │
│ リード獲得   │───→│ リード精査   │───→│ 商談・提案   │───→│ 導入支援     │
│ コンテンツ   │    │ ヒアリング   │    │ デモ         │    │ 活用促進     │
│ 広告・SEO    │    │ ニーズ確認   │    │ 見積・交渉   │    │ 拡大提案     │
│ セミナー     │    │ 商談設定     │    │ 契約締結     │    │ 解約防止     │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
       │                   │                   │                   │
   リード数            商談化率             受注率            継続率・LTV
```

### ■ 重要な概念

| 概念 | 内容 |
|------|------|
| **MQL → SQL** | マーケティングが作ったリード（MQL）を、インサイドセールスが精査して「商談に値する」と判断したもの（SQL）に変換する |
| **パイプライン管理** | 商談の進捗を可視化し、各フェーズのボトルネックを特定して改善する |
| **The Model = 分業×連携** | 単なる分業ではなく、各チーム間のデータ連携と引き継ぎルールが肝 |
| **カスタマーサクセス** | 売って終わりではなく、顧客の成功を支援することで解約防止とアップセルを実現 |

### ■ HARMONIC insight への示唆

| プロセス | 現状の対応 | 改善の余地 |
|---------|-----------|-----------|
| **マーケティング** | Webサイト4つ、FPTチャネル | コンテンツマーケティング（ブログ・事例・ホワイトペーパー）の強化 |
| **インサイドセールス** | IMA無料診断がリード精査の役割を果たしている | 診断結果から商談への転換プロセスを明確化・数値管理 |
| **フィールドセールス** | FPT経由の場合はFPTが担当 | 直販チャネル（InsightCast等）の商談プロセスを設計 |
| **カスタマーサクセス** | SaaSモデルでの継続支援 | 導入後の活用支援プログラム、定期レビュー、アップセル提案の仕組み化 |

## 6冊の横断まとめ ── 今後のアクションへの統合

### 共通して言えること

| テーマ | 6冊を通じた共通メッセージ |
|--------|-------------------------|
| **顧客起点** | すべての施策は「顧客は誰か」「何に困っているか」から始まる |
| **仕組み化** | 属人的な営業ではなく、再現可能なプロセスとして設計する |
| **段階設計** | いきなり売るのではなく、小さなYesから大きなYesへ段階的に導く |
| **証拠と信頼** | 実績・事例・第三者の声が最も強い説得力を持つ |
| **テストと改善** | 小さく試して、数値で判断し、勝ちパターンに集中する |

### HARMONIC insight の「次にやるべきこと」優先順位案

```
【即実行（今週〜）】
  ├─ ① USPの再整理と全チャネルでの統一訴求（①②より）
  │    → 「世界唯一のBizRobo→aKaBot移行」を全タッチポイントで一貫発信
  │
  ├─ ② 導入事例・成功実績の整備（⑤③より）
  │    → 社会的証明と権威の構築。具体的な数字入りの事例を作成
  │
  └─ ③ IMA無料診断→商談の転換率を計測開始（⑥①より）
       → 現状の数値を把握し、改善のベースラインを確立

【短期（1ヶ月以内）】
  ├─ ④ 各製品LPの再構成（③より）
  │    → 「問題提起→解決策→証拠→CTA」のフォーマットに統一
  │
  ├─ ⑤ 紹介プログラムの設計（①④より）
  │    → 既存顧客・パートナーからの紹介を仕組み化
  │
  └─ ⑥ BtoB営業プロセスの4分業設計（⑥より）
       → マーケ→IS→FS→CSの流れと各KPIを定義

【中期（3ヶ月以内）】
  ├─ ⑦ コンテンツマーケティング開始（②⑥より）
  │    → ブログ・ホワイトペーパー・セミナーでリード獲得
  │
  ├─ ⑧ Web広告のABテスト運用開始（③より）
  │    → InsightCast向けに少額から広告テストを回す
  │
  └─ ⑨ カスタマーサクセス体制の構築（⑥より）
       → 導入後の活用支援→アップセル→紹介の好循環を作る
```
