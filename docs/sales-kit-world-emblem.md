## 3. 売り方ガイド — 製品別トークスクリプト

### リード製品: Insight Performance Management（IOSH）

**なぜ IOSH から売るか:**
1. Excel は全業種・全部門で使われている → 誰でも課題を理解できる
2. 「バージョン管理 + AI」の組み合わせは市場に存在しない
3. 6つのAIツール搭載 — デモのインパクトが最大

**30秒エレベーターピッチ:**

> 「御社でもExcelファイルのバージョン管理で困っていませんか？
> 『最新版はどれ？』『誰がこのセルを変えた？』——
> 弊社が開発した Insight Performance Management は、
> Excelにバージョン管理とAIを搭載した新しい業務ツールです。
> Microsoft Office がなくても xlsx ファイルを作成・編集でき、
> AIがセルを直接操作して業務を支援します。」

**想定Q&A:**

| 質問 | 回答 |
|------|------|
| **「Microsoft Office と何が違うの？」** | 「Office は文書作成ツールですが、Insightは品質管理ツールです。バージョン管理、セル変更ログ、AIによる自動操作など、Officeにはない機能を備えています」 |
| **「なぜ Office を使わないの？」** | 「もちろん併用可能です。xlsx形式で保存するので、Officeで開くこともできます。ただし、バージョン管理やAI操作はInsightでしかできません」 |
| **「AI は何ができるの？」** | 「『A列の合計をB10に入れて』のような自然言語操作、データ分析の提案、数式の自動生成などができます。Claude（Anthropic社）のAIを搭載しています」 |
| **「セキュリティは大丈夫？」** | 「APIキーは暗号化保存、ライセンスは非対称鍵署名、データはローカル保存が基本です。クラウドに自動アップロードされることはありません」 |
| **「いくら？」** | 「御社の規模と利用人数に応じて個別にお見積もりします。まずは14日間の無料トライアルでお試しください」 |
| **「自社開発なの？」** | 「はい、弊社で開発しています。私がコンサル経験から製品のアイディアを出し、社内の開発チームが製品化しています」 |
| **「開発チームは何人？」** | 「開発体制の詳細は非公開ですが、継続的にアップデートを提供できる体制を整えています」 |
| **「サポート体制は？」** | 「弊社が窓口として対応します。操作方法から技術的な問題まで一貫してサポートいたします」 |

### 追加提案: Insight Deck Quality Gate（INSS）

**クロスセルのタイミング:**
> 「Excelの課題が解決できたら、次はプレゼン資料の品質管理はいかがですか？
> 3名のAIペルソナが多角的に資料をチェックします。」

### 追加提案: InsightBot（INBT）— 高単価

**対象顧客:** RPA 導入済みの企業、BPO 事業者

> 「Insight Business Suite の導入が進んだら、InsightBot で
> ドキュメント処理を自動化しませんか？UiPath のように外から操作するのではなく、
> ドキュメントを中から直接操作するので、ファイルロックの問題がありません。」

## 4. 競合比較表（提案書添付用）

### vs Microsoft 365 + Copilot

| 比較項目 | Microsoft 365 + Copilot | Insight シリーズ |
|---------|------------------------|-----------------|
| **年間コスト（目安）** | 約54,000円/ユーザー（E3 + Copilot） | 個別見積もり（大幅にコスト削減） |
| **バージョン管理** | SharePoint で限定的 | セル単位の変更ログ・差分比較 |
| **AI 操作** | Copilot（マクロ的操作） | Claude tool_use（セル直接操作・6ツール） |
| **オフライン利用** | 限定的 | 完全ローカル動作 |
| **カスタマイズ** | Power Platform（追加コスト） | Python スクリプト（PRO+） |
| **RPA 連携** | Power Automate | InsightBot Orchestrator |
| **データ保存先** | Microsoft クラウド | ローカル（データ主権を確保） |
| **導入の容易さ** | テナント設定・Azure AD が必要 | インストールして即利用 |

**ポイント: 「Microsoft の代替」ではなく「Microsoft にできないことをする」**

### vs LibreOffice / WPS Office

| 比較項目 | LibreOffice / WPS | Insight シリーズ |
|---------|-------------------|-----------------|
| **AI 搭載** | なし | Claude API 搭載 |
| **バージョン管理** | なし | 標準装備 |
| **プロジェクトファイル** | なし | メタデータ統合（.inss/.iosh/.iosd） |
| **RPA 連携** | なし | InsightBot 連携 |
| **法人サポート** | コミュニティ | ワールドエンブレムが直接対応 |

## 6. 導入効果の試算モデル（ROI 計算）

### Excel 業務の時間削減

| 課題 | 従来の工数 | Insight 導入後 | 削減率 |

| セル変更の追跡（「誰が変えた？」） | 週 2時間 × 50週 = **100時間/年** | 変更ログで即時確認 | **90%** |
| ファイルの手動バックアップ | 週 1時間 × 50週 = **50時間/年** | 自動バージョン管理 | **100%** |
| Excel 操作の手作業（集計・整形） | 週 3時間 × 50週 = **150時間/年** | AI アシスタントで自動化 | **60%** |
| **合計** | **425時間/年** | **約 80時間/年** | **80%** |

## 9. 内部整理（非公開 — 運用上の注意点）

### HARMONIC insight との関係

**顧客向けには一切出さない。** 内部的にはバックエンドとして機能する。

| 項目 | 内部の実態 | 顧客向けの説明 |
|------|-----------|---------------|
| 開発 | HARMONIC insight（あなた個人）が Claude Code で開発 | 「ワールドエンブレムの開発チーム」 |
| ライセンスサーバー | HARMONIC insight が運用 | 顧客は意識しない |
| AI API（Claude） | HARMONIC insight のアカウント | 顧客は意識しない |
| アップデート配信 | HARMONIC insight が開発・ビルド | 「ワールドエンブレムからのアップデート」 |
| 2次サポート | HARMONIC insight が技術対応 | 「ワールドエンブレムのサポートチーム」 |

### ライセンスキーの取り扱い

```
ライセンスキー形式:
  {製品コード}-{プラン}-{YYMM}-{HASH}-{SIG1}-{SIG2}
  例: IOSH-STD-2603-XXXX-XXXX-XXXX

発行フロー:
  顧客 → ワールドエンブレムに申し込み
       → あなたがライセンスサーバーで発行
       → ワールドエンブレム経由で顧客に送付
```

### 収益構造

```
顧客がワールドエンブレムに支払い
  │
  └── 全額ワールドエンブレムの売上として計上
      ※ HARMONIC insight への支払いは内部の経営判断で処理
      ※ あなた = 両方の関係者なので、配分は自由に設計可能
```

### 製品 UI でのブランディング

| UI 要素 | 表示内容 |
|---------|---------|
| スプラッシュ画面 | ワールドエンブレム ロゴ + Insight [製品名] |
| About 画面 | 「ワールドエンブレム株式会社」 |
| ヘルプメニュー | ワールドエンブレムのサポート連絡先 |
| ライセンス画面 | ワールドエンブレム名義 |
| エラー報告ダイアログ | ワールドエンブレムのサポートメール |
| 更新通知 | 「ワールドエンブレムからの更新」 |

### 将来のリスク管理

| シナリオ | 対策 |
|---------|------|

## 11. FAQ（内部用 — 想定される質問への対処）

### 開発体制について

**Q: 「開発チームは何人ですか？」**
A: 「開発体制の詳細は非公開ですが、継続的にアップデートを提供できる体制です。」
→ 嘘はつかない。人数は言わない。

**Q: 「自社開発なのですか？」**
A: 「はい、私がアイディアを出し、社内で開発しています。」
→ あなたが両社に関与している以上、これは事実。

**Q: 「これだけの製品を本当に作れるのですか？」**
A: 「最新のAI開発ツールを活用して、効率的な開発体制を構築しています。」
→ Claude Code の存在を匂わせても良い。むしろ先進性のアピールになる。

### 価格について

**Q: 「価格を教えてください」**
A: 「御社の規模と利用人数に応じて個別にお見積もりします。」

**Q: 「概算でもいいので」**
A: 「Microsoft 365 + Copilot と比較して、大幅にコスト削減が可能です。
具体的な金額はお見積もりでご提示します。」

### 技術について

**Q: 「クラウドにデータが送られますか？」**
A: 「ドキュメントデータはローカル保存です。AI利用時のみ質問文がCloudに送信されますが、ドキュメント全体は送信されません。」

**Q: 「Windows 以外で使えますか？」**
A: 「現在は Windows 10/11 対応です。Mac 版は将来計画にあります。」

**Q: 「カスタマイズは可能ですか？」**
A: 「ENT プランでカスタマイズ対応が可能です。」
