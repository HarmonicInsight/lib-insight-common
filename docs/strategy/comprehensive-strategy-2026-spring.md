## 2. 二刀流戦略の再設計: FPT MD × HARMONIC insight

### 2.1 根本原則: 「二人の人格」ではなく「一人のストーリー」

既存戦略では「交わらないが共鳴する」としていたが、これを **一段階進化** させる。

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│   旧モデル: 「2つの顔を隠す」                                       │
│   ───────────────────────────                                       │
│   FPTのMD ←── 壁 ──→ HIの代表                                      │
│   問題: 壁を維持するコストが高い。説明に矛盾が生じやすい。           │
│   実態: 日本の合同会社登記は公開情報。隠すことは現実的に不可能。     │
│                                                                     │
│   新モデル: 「オープンな二足のわらじ」                               │
│   ─────────────────────────────────                                 │
│   「コンサルタントとしての課題意識が、製品開発の原動力」             │
│                                                                     │
│   FPT MD としての発信                HIの代表としての実行            │
│   ┌────────────────────┐            ┌────────────────────┐         │
│   │ 「業界の課題を語る」 │            │ 「課題を解決する」  │         │
│   │  思想家・実務家      │ ──同一人物→ │  開発者・起業家      │         │
│   │  として信頼を構築    │            │  として製品を提供    │         │
│   └────────────────────┘            └────────────────────┘         │
│                                                                     │
│   ★ 鍵: FPTの業務でHI製品を「押し売り」しない。                      │
│         しかし「自分が開発に関わっている」こと自体は隠さない。       │
│         利益相反を避けるのは「行動」であって「情報の隠蔽」ではない。 │
│                                                                     │
│   ★ なぜこのモデルが優れているか:                                    │
│     1. 隠すことによる信頼リスクがゼロ                                │
│     2. 「コンサルが自分の課題を自分で解決した」は最強のストーリー    │
│     3. 日本の起業文化では副業・兼業が急速に社会的に許容されつつある  │
│     4. FPTが外資系であることが有利（外資は副業に寛容な傾向）        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 二刀流の実行ルール

| ルール | 具体的な行動 | 違反例（やってはいけない） |
|--------|-------------|--------------------------|
| **R1: 知見は共有、製品は分離** | FPTの名刺で「AI×業務効率化の知見」を語る。HIの名刺で「製品の導入」を提案する | FPTの商談でHI製品の見積もりを出す |
| **R2: FPT顧客へは第三者導入** | FPT案件でHI製品を使う場合、FPTのデリバリーチームが評価・選定する形にする | 自分が直接FPTクライアントにライセンスを販売する |
| **R3: 実績は事実だけ述べる** | 「FPT Consulting Japanで評価中」「社内パイロット実施」は事実として使える | 「FPT Consulting推奨ツール」と謳う |
| **R4: LinkedInは個人ブランド** | 個人の見識として「AI×コンサル」を発信。HIの製品紹介はHIの公式アカウントから | FPTの肩書きでHI製品を宣伝するLinkedIn投稿 |
| **R5: 利益相反の明示的管理** | FPTでHI製品の評価を推進する場合、利害関係者に開示する | 自分が開発者であることを隠してFPT内で推進する |

### 2.3 「オープンな二足のわらじ」が生むナラティブ

```
LinkedIn投稿例（FPTの個人アカウント）:

「コンサルタントとして28年。最大のフラストレーションは、

 この課題を自分で解決しようと、
 HARMONIC insight という会社で AI Office ツールを作り始めて

 コンサルだから見える課題。エンジニアだから作れる解決策。
 この2つが揃ったとき、製品は本当に「使える」ものになる。

 #AI #コンサルティング #生産性」

→ これは「広告」ではない。「ストーリー」。
→ Seth Godin: "Marketing is no longer about the stuff you make,
   but about the stories you tell."
```

### 3.2 Engine 1: Insight Business Suite ── フロントドア戦略

#### なぜInsightSheetが入口か（7冊の理論が一致する結論）

```
① 非常識なマーケティング大全 → 「教育フェーズ」に最適

   → この一言で「悩みの自覚」を促せる（教育→販売の2ステップ）

   BIZ版は個別見積もり。FREE版で価値を体験してから購入検討
```

## 4. 「教育」フェーズの設計 ── 7冊の理論が指摘する最大の欠落

```

  全製品共通で「教育フェーズが完全に欠落」。
  集客 → (教育なし) → 販売 → リピート
  という構造になっている。

  小山竜央: 「人がモノを買うプロセスは、教育→販売の2ステップ」
  Jay Abraham: 「顧客を教育してからオファーする」

  → 3人の理論が同じことを言っている:
     「いきなり売るな。まず教育しろ」
```

```
Stage 1: 「問題の自覚」（SEO + ブログ + YouTube）
```

## 8. リスク管理と対策

### 8.1 最大のリスク: 「何もかもやろうとして何も達成しない」

```
5製品 × 3エンジン × 2つの顔 × マーケティング施策
= 組み合わせが膨大すぎて、一人では実行しきれない

対策: 「90日間は InsightSheet FREE版の公開だけに集中する」

  理由:
  1. FREE版は「ビルドするだけ」で公開できる（追加開発ほぼ不要）
  2. SEO記事は1本書けば24時間365日集客し続ける
  3. ダウンロード数という定量的KPIが得られる
  4. FREE → BIZ の転換率で「市場の反応」がわかる
  5. 他の施策はFREE版の結果を見てから判断できる

  → 最初の一手 = InsightSheet FREE版公開
  → これだけは3月第2週までに必ず実行する
```

### 8.2 リスク一覧と対策

| リスク | 確率 | 影響 | 対策 |
|--------|------|------|------|
| **FPTとの利益相反** | 中 | 致命的 | 「オープンな二足のわらじ」モデルに移行。隠蔽ではなく適切な開示と行動の分離で管理 |
| **Copilotの急速な進化** | 高 | 高 | 「Copilotが入れない場所」に特化。ローカル完結・オフライン対応を堅持。政府・自治体・金融を重点ターゲット |
| **開発リソースの分散** | 高 | 高 | InsightSheet FREE版 → Suite バンドル → InterviewInsight の順で集中。同時に3製品以上に手を出さない |
| **集客の失敗（SEO不発）** | 中 | 中 | 「Excel バージョン管理」のキーワードボリュームを事前に確認。不発ならLinkedIn + 紹介に切り替え |
| **Claude API コスト** | 低 | 中 | 3人格モデル（Haiku/Sonnet/Opus）で最適化済み。Haiku利用率を高めるUXを維持 |
| **Syncfusion ライセンスコスト** | 低 | 低 | Community License（年商$1M未満）が利用可能。$1M超えた時点で切り替え判断 |

## 9. InsightDoc の戦略的位置づけ

### 9.1 現実的な評価

```
marketing-theory-check-5products.md:
  InsightDoc の全製品評価: D（最低）
  ・差別化: ×（Word + OneDrive で十分では？）
  ・集客: ×（チャネルなし）
  ・USP: 不明確

コード精査の結果:
  ・CLAUDE.md が insight-common の説明のみ（製品独自のガイドなし）
  ・publish/ フォルダにビルド済みバイナリが直接コミット（86MB）
  ・機能: 参照資料連携 + AI統合 + バージョン管理

結論:
  InsightDoc は「単体で売れる製品」ではない。
  しかし「Suite のバンドル商材」としては十分な価値がある。

  → InsightDoc の単体サイト・単体販売は行わない
  → Suite (Slide + Doc + Sheet) の一部としてのみ提供
  → 「3製品セット」のお得感を演出する役割
```

### 9.2 InsightDoc の改善優先度

| 改善項目 | 優先度 | 理由 |
|---------|--------|------|
| publish/ からバイナリを除去し、CI/CD でビルド | 高 | リポジトリの品質管理 |
| 製品固有の CLAUDE.md を作成 | 中 | 開発効率の向上 |
| 独自の差別化機能を追加 | **低** | Suite バンドルの一部なので単体の差別化は不要 |
