## ① 小山竜央『非常識なマーケティング大全』で検証

### コア原則チェック

```
5つのコア原則:
  1. 集客第一主義
  2. 差別化とブランディング
  3. 仕組み化による再現性
  4. 心理とコピー技術
  5. SNS・YouTube活用

ビジネスの4つの流れ:
  集客 → 販売 → サービス提供 → リピート化
```

#### 原則1: 集客第一主義 ── 「売ること」ではなく「集めること」が原点

| 製品 | 判定 | 状況 |
|------|------|------|
| Cast | △ | 自己証明マーケティング（自社製品で動画を作って見せる）は構想にあるが、**まだ実行されていない**。動画を作って集客する「仕組み」が未稼働 |
| Slide | × | 集客チャネルが未設計。「コンサル先に入れる」だけで、新規リードを集める仕組みがない |
| Sheet | △ | 「Excelバージョン管理 無料」でSEO流入を狙う構想はあるが、未実行 |
| Doc | × | 集客の設計がゼロ。製品の存在を知らせる手段がない |
| Interview | △ | VRMアバター対話型という珍しさで話題性はあるが、それを集客に活かす施策が未設計 |

**小山理論の指摘:**
> 「良い商品を持っていても、お客様を集められなければビジネスは成立しない」

→ 5製品とも「製品はある」が「集客の仕組み」がない。コンサル紹介に頼っているが、
  それは「集客」ではなく「紹介待ち」。紹介を「仕組み化」する設計が欠落。

#### 原則2: 差別化 ── 「競合他社が言わないことを言う」

| 製品 | 判定 | USP（他社が言えないこと） | 問題点 |
|------|------|--------------------------|--------|
| Cast | ○ | 「VoiceVox + FFmpeg = 限界費用ゼロ。1本¥596」 | USPは明確。ただし顧客にこの言葉が届いていない |
| Slide | △ | 「3人格AI（Haiku/Sonnet/Opus）でコスト意識と品質を両立」 | 差別化はあるが「誰にとって何が嬉しいか」が曖昧 |
| Sheet | △ | 「Excel + AI Tool Use + バージョン管理」 | Microsoft Copilotとの差が顧客目線で見えにくい |
| Doc | × | 「参照資料連携 + バージョン管理」 | 差別化が弱い。Word + OneDriveで十分では？という疑問に答えられない |
| Interview | ○ | 「VRMアバターとの対話型。100人同時。フォーム型とは根本的に違う」 | カテゴリー自体が新しい。カテゴリーキング戦略が取れる |

**小山理論の指摘:**
> 「商品を変えなくても、伝え方・切り口を変えるだけで差別化は可能」

→ Cast と Interview は強い差別化がある。
→ Sheet は Copilot との比較で「何が違うのか」を明確に言語化すべき。
→ Doc は差別化そのものを再構築する必要がある。

#### 原則3: 仕組み化 ── 偶然ではなく設計でヒットを必然にする

| 製品 | 判定 | 状況 |
|------|------|------|
| 全製品共通 | × | マーケティングファネル（集客→教育→販売→リピート）が設計されていない |

```
現状のファネル:

  集客:      コンサル紹介（属人的）
     ↓
  教育:      なし（FREE で触るだけ）
     ↓
  販売:      TRIAL 発行 → 稟議 → BIZ/ENT
     ↓
  リピート:   自動更新（仕組みはある）

問題点:
  ・「教育」フェーズが完全に欠落
  ・小山理論: 「人がモノを買うプロセスは、教育 → 販売 の2ステップ」
  ・教育 = 「悩みを自覚していない人に、悩みを自覚させる」
  ・今は「すでに悩みを自覚している人」にしか売れない構造

あるべき姿:
  集客:      YouTube動画・ブログ・紹介プログラム
     ↓
  教育:      「Excelのバージョン管理、まだ手動？」
             「動画制作、まだ外注に¥300,000？」
             「100人ヒアリング、まだ手動で250時間？」
     ↓
  販売:      FREE → TRIAL → BIZ/ENT
     ↓
  リピート:   カスタマーサクセス + クロスセル
```

#### 原則4: 心理とコピー技術

| 製品 | 判定 | 現状のコピー | 改善案 |
|------|------|-------------|--------|
| Cast | ○ | 「1本¥596、外注の500分の1」 | 数字が明確で強い。このまま使える |
| Slide | × | コピーなし | 「150枚のPPT、手動レビューなら2日。InsightSlideなら30分」 |
| Sheet | × | コピーなし | 「report_v2_final_FINAL.xlsx、まだやってますか？」 |
| Doc | × | コピーなし | 未設計。製品の存在意義から再定義が必要 |
| Interview | △ | 「ROI 7.5倍」 | 「100人のヒアリング、手動なら250時間。InterviewInsightで全自動」 |

**小山理論の指摘:**
> 「初心者を囲い込む最強キラーワード」「目的別コピーライティング28選」

→ 5製品中、コピーが完成しているのは Cast のみ。
→ 他4製品は「何を言うか」が未定義。製品を作る前にコピーを書くべきだった。

#### 原則5: SNS・YouTube活用

| 判定 | 状況 |
|------|------|
| △ | YouTube 6本以上の計画はあるが未実行。SNS 3-5投稿/週の計画も未実行 |

**小山理論の指摘:**
> 「チャンネルを増やすだけでなく育てる視点が重要」

→ 計画はあるが1本も実行されていない。「仕組み化」以前の問題。

#### 第2章の重要概念: イノベーター理論

```
小山理論が引用するイノベーター理論（ロジャーズ教授）:

  イノベーター（2.5%）    → 新しいもの好き。放っておいても買う
  アーリーアダプター（13.5%）→ 先見性がある。良ければ広めてくれる
  アーリーマジョリティ（34%）→ 慎重。実績を見てから買う
  レイトマジョリティ（34%） → さらに慎重。みんなが使ってから買う
  ラガード（16%）          → 変化を嫌う。必要に迫られて買う

現在のターゲット:
  コンサルの既存顧客 = すでに信頼関係がある
  → イノベーター〜アーリーアダプターに位置する人たち
  → この層は「教育」なしでも買ってくれる可能性がある

しかし:
  紹介チャネルの先にいる人 = アーリーマジョリティ
  → この層には「教育」（= 悩みを自覚させること）が必須
  → 今の設計では紹介チャネルが詰まる
  → 紹介されても「よくわからない」→ 購入しない

対策:
  ・各製品の「教育コンテンツ」を用意する（動画、記事、事例）
  ・紹介者が「この URL を送って」で済むようにする
  ・URL の先に「悩みの自覚 → 解決策の理解 → 行動」の流れがある
```

## ② ジェイ・エイブラハム『ハイパワー・マーケティング』で検証

### 核心フレームワーク: 売上 = 顧客数 × 客単価 × 購買頻度

#### レバー1: 顧客数を増やす

| 施策 | 判定 | 状況 |
|------|------|------|
| **紹介プログラム** | × | 「知り合いのコンサルに紹介してもらう」とあるが、仕組み化されていない。紹介インセンティブなし、紹介用URL/ツールなし |
| **ジョイントベンチャー** | ○ | FPTチャネルは典型的なJV。ただしFPTチャネルが5製品すべてに適用できるかは未確認 |
| **リスクリバーサル** | △ | FREE + TRIAL は存在するが、「成果保証」「返金保証」は未設計 |
| **USP** | △ | Cast と Interview は明確。Sheet/Slide/Doc は不十分 |

```
エイブラハムの紹介プログラム設計:

  1. 既存顧客に「紹介しやすい仕組み」を提供する
     → 紹介用URL、紹介カード、メールテンプレート
  2. 紹介者にインセンティブを与える
     → 売上の20%、無料期間延長、機能アップグレード
  3. 被紹介者にもインセンティブを与える
     → 初月50%OFF、無料トライアル延長

  現状: これらが一切ない
  → コンサル紹介は善意に依存。善意は持続しない。
```

#### レバー2: 客単価を上げる

| 施策 | 判定 | 状況 |
|------|------|------|
| **アップセル** | ○ | FREE → BIZ → ENT の段階設計は存在。Suite バンドルもある |
| **クロスセル** | △ | 5製品のクロスセル設計があるが、「どの順番で、いつ提案するか」が未定義 |
| **価格の再設定** | ○ | 価値ベースの再設計が済んでいる（pricing-licensing-strategy-2026.md） |

```
エイブラハムのクロスセル設計:

  「関連商品をセットで提案する」

  提案: 5製品のクロスセルマトリクスを定義する

  Sheet を買った顧客 → 次に提案すべきは？
    ・Doc（同じOffice系。自然な拡張）       → 1ヶ月後に提案
    ・Slide（提案資料のレビュー）            → 2ヶ月後に提案
    ・Suite バンドル割引                     → 3ヶ月後に提案
    ・Interview（別カテゴリだが同じ顧客層）  → 6ヶ月後に提案
    ・Cast（コンテンツ制作）                 → 必要に応じて

  今これが明文化されていない。
  コンサルとして訪問した際に「ところで他にもあります」では戦略ではない。
```

#### レバー3: 購買頻度を上げる

| 施策 | 判定 | 状況 |
|------|------|------|
| **定期接触** | × | メルマガ、ニュースレター等の定期接触の仕組みがない |
| **サブスクリプション** | ○ | 年額/月額課金で自動更新。構造的に解決 |
| **期限付きオファー** | × | 「早期採用者特別価格」は計画にあるが未実行 |

```
エイブラハムの「卓越の戦略」:

  「顧客を取引相手ではなく、守るべき存在と捉える」

  → コンサル業では自然にこれをやっている（良い点）
  → ただし、製品販売になると「売って終わり」になりがち
  → カスタマーサクセス（導入後の成功支援）が未設計
```

**エイブラハム理論での最大の問題:**
> 紹介プログラムの仕組み化が最優先。コンサル紹介は最強のチャネルだが、
> 善意だけでは持続しない。インセンティブと仕組みが必要。

## ⑤ 竹内謙礼『売り方の正解』で検証

### 小規模事業者として使えるヒント

```
本書の立場: 「人も金も時間もない状況で、明日からすぐ試せる施策」
→ 1人会社の HARMONIC insight にはこの視点が最も実用的
```

## ⑦ 福田康隆『THE MODEL』で検証

### 4プロセスチェック

```
マーケティング → インサイドセールス → フィールドセールス → カスタマーサクセス
```

| プロセス | 判定 | 現状 | 問題点 |
|---------|------|------|--------|
| **マーケティング** | △ | Webサイト4つ、YouTube計画、PR計画 → ただし未実行 | リード獲得の仕組みが稼働していない |
| **インサイドセールス** | × | 該当なし（1人会社なのでISの分業不可） | ただしコンサル紹介がISの代替機能を果たしている |
| **フィールドセールス** | ○ | コンサル訪問時に直接提案 → FSそのもの | 1人でFSを回せる範囲は月5-10社が限界 |
| **カスタマーサクセス** | × | 導入後の支援プログラムなし | 解約防止・アップセル・クロスセルの仕組みがない |

```
THE MODEL を1人会社に適用する場合の再解釈:

  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
  │マーケティング │    │インサイド     │    │フィールド     │    │カスタマー     │
  │              │    │セールス       │    │セールス       │    │サクセス       │
  │              │    │              │    │              │    │              │
  │ YouTube      │    │ 紹介者が      │    │ 自分が        │    │ 自分が        │
  │ ブログ       │───→│ フィルタリング│───→│ コンサル訪問  │───→│ 定期フォロー  │
  │ SEO          │    │ してくれる    │    │ で直接提案    │    │ + クロスセル  │
  │ 紹介プログラム│    │              │    │              │    │              │
  └──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
         │                   │                   │                   │
     リード数            紹介成約率           提案成功率          継続率・LTV

  現状:
  ・マーケティング = 未稼働
  ・IS = 紹介者任せ（仕組み化されていない）
  ・FS = 動いている（唯一の強み）
  ・CS = 未設計

  → 4プロセス中、1つしか機能していない
```

#### THE MODEL の MQL → SQL 概念の適用

```
1人会社版 MQL → SQL:

  MQL（Marketing Qualified Lead）
  = 「製品に興味を持った人」
  = FREE版をダウンロードした人、紹介された人、セミナー参加者

  SQL（Sales Qualified Lead）
  = 「予算・権限・ニーズがある人」
  = TRIAL を使い始めた人、コンサル訪問で具体的に話が進んだ人

  現状の問題:
  ・MQL を生む仕組みがない（マーケティング未稼働）
  ・MQL → SQL の判定基準がない
  ・SQL → 受注のプロセスはコンサル訪問で成立（OK）

  KPI を設定すべき:
  ・月間 MQL 数: ○○件（目標: 月20件）
  ・MQL → SQL 転換率: ○○%（目標: 30%）
  ・SQL → 受注率: ○○%（目標: 50%）
  ・月間受注数: ○○件（目標: 3件）
```

```
指摘する理論:

### 問題2: 社会的証明がゼロ 【深刻度: ★★★★★】

```
指摘する理論:

  → 社会的証明がないと紹介が成約しない

### 全7冊が共通して指摘する最大の欠落
