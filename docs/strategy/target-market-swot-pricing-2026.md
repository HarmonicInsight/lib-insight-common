## 1.3 ターゲットセグメント詳細

### プライマリーターゲット（最優先で攻める市場）

#### Segment A: M365未導入の中小企業（従業員10-300名）

```
プロファイル:
  ・業種: 製造、建設、小売、サービス、士業
  ・IT環境: Windows PC + Excel/Word/PowerPoint（買い切り版）
  ・課題: 「AIを使いたいがM365は高い/クラウドは不安」
  ・意思決定: 部門長 or 社長が直接判断
  ・予算感: 年間5万円以下なら稟議不要

なぜこのセグメントか:
  ① 358万社の大半がここに属する（圧倒的な数）
  ② Copilotが届かない（M365前提のため）
  ③ InsightSheet BIZ は個別見積もりで柔軟な価格設定
  ④ 「Excelのバージョン管理」は普遍的な痛み
  ⑤ SEO「Excel バージョン管理」で自然流入が狙える

攻め方:
  InsightSheet FREE版（全機能・保存不可） → SEO流入 → BIZ購入 → Suite アップセル
```

#### Segment B: コンサルティングファーム（従業員50-5,000名）

```
プロファイル:
  ・業種: 経営コンサル、ITコンサル、SIer
  ・IT環境: M365導入済みだがCopilotは未導入/パイロット中
  ・課題: 「ヒアリング→提案書→レビューの工数が膨大」
  ・意思決定: パートナー/ディレクター層
  ・予算感: 月額5-10万円なら部門予算で承認可能

なぜこのセグメントか:
  ① あなた自身がFPT MDとして内部にいる（市場へのアクセスが完璧）
  ② InterviewInsight + InsightSlide のクロスセルが自然
  ③ コンサルは「提案書とヒアリング」が業務の核心
  ④ 高単価（InterviewInsight BIZ 個別見積もり）を受容できる
  ⑤ FPT社内導入で「大手コンサルファーム導入実績」が作れる

攻め方:
  FPT社内パイロット → 実績化 → LinkedIn発信 → 他ファームへ展開
```

## 2.2 クロスSWOT戦略

### SO戦略（強み × 機会）── 攻めの戦略

| # | 強み | 機会 | 戦略 |
|---|------|------|------|
| **SO1** | S1 統一基盤 + S6 AI BYOK | O1 中小企業95%がAI未導入 | **「AI搭載・全機能無料体験」でM365未導入の中小企業を一気に取り込む。** AI導入の最大障壁「使い方がわからない」を3人格AI（S7）で解消し、「AI初体験」としてInsightSheetを位置づける |
| **SO2** | S2 FPT MDアクセス | O6 FPT社内導入 | **FPT社内パイロットで「大手コンサルファーム導入実績」を最速で構築。** この実績は全製品の営業活動に横展開可能な最強の社会的証明になる |
| **SO3** | S3 AI Tool Use | O5 Excel SEO需要 | **「Excel バージョン管理 無料」でSEO 1位を獲得し、InsightSheet FREE版への流入を最大化。** AI Tool Useの体験がアップセルの決め手になる |
| **SO4** | S5 ローカル完結 | O4 Copilot不可市場 | **「Copilotが入れない場所に、AIを。」のメッセージで政府・金融・医療を攻める。** セキュリティ要件が厳しい市場はローカル完結型の独壇場 |
| **SO5** | S4 限界費用ゼロ | O2 補助金 | **Insight Training Studio を「デジタル化・AI導入補助金」対象ツールとして申請。** 限界費用ゼロなので補助金で初年度無料に近い体験を提供しても利益が出る |
| **SO6** | S8 InterviewInsight完成度 | O7 新カテゴリー | **「AI対話型ナレッジキャプチャ」というカテゴリーを定義し、カテゴリーキングを狙う。** SurveyMonkey等のフォーム型とは競合しない新市場を創造する |

```
  → 価格は「安い」のではなく「コスパが高い」
```

### 3つの価格設計原則（再確認・進化版）

```
原則1: 「AIは全プラン BYOK。制限なし。」
  ─────────────────────────────────────
  ユーザーが自身の Claude API キーを持ち込む。
  モデル制限・回数制限なし。HARMONIC側のAIコスト負担=ゼロ。
  HARMONIC の役割 = 最大限効率化できる仕組みを作ること。

  根拠:
  ・APIコストはユーザー負担 → HARMONIC側のマージンリスクがゼロ
  ・AI別売りやティア制限は2026年では時代遅れ
  ・「AIを使うたびにチケットが減る」心理的ストレスを完全排除
  ・FREE でも AI が使える → 製品の価値を100%体験できる

原則2: 「入口は全機能無料、保存で課金」
  ──────────────────────────────────────
  FREE = 全機能利用可能（AI含む）、ただし保存・エクスポート不可。
  BIZ = 保存・エクスポート可能（個別見積もり）。
  ENT = BIZ + カスタマイズ（SSO・監査・API等）。

  根拠:
  ・全機能を体験させてから課金 → 損失回避バイアスが発動
  ・TRIAL = BIZ 同等を無料で配布 → 導入障壁ゼロ
  ・ENT = 企業管理機能のカスタマイズ → IT部門の要件に対応

原則3: 「価値の10%を価格とする」
  ──────────────────────────────
  コスト積み上げではなく、顧客が得る価値から逆算。
  日本市場では「得られる価値の10%」が許容される価格感覚。

  根拠:
  ・InsightSheet: 年間価値 ¥523,000 → 個別見積もり
  ・InterviewInsight: 月間価値 ¥300,000 → 個別見積もり
  ・Insight Training Studio: 月間価値 ¥3,000,000 → 個別見積もり
    → 全製品BIZ/ENTは個別見積もり。顧客規模に応じた柔軟な価格設定
```

## 3.4 価格心理学の適用

### アンカリング効果

```
LP上での価格表示の設計:

  「Microsoft 365 + Copilot: 年間 ¥76,800/人」
  ────────────────────────────────────────────

  → 「/人」vs「/PC」の違いも明示（PCは共有可能な場合あり）
```

### デコイ効果（おとり効果）

```
InsightSheet の3プランの設計:

  FREE: ¥0（全機能・保存不可）       ← 入口（全機能体験）
  BIZ:  個別見積もり（保存・エクスポート可） ← 顧客規模に応じた柔軟設定
  ENT:  個別見積もり（+カスタマイズ）  ← 企業管理

  → FREE → BIZ/ENT のステップアップ（TRIAL = BIZ 無料配布）
  → 個別見積もりにより顧客ごとの最適な価格設定が可能

Insight Business Suite のバンドル設計:

  Sheet BIZ 単体: 個別見積もり
  Suite BIZ (3製品): 個別見積もり

  → 「Sheet単体よりSuiteの方がお得」を見積もり時に提示
  → バンドルの「お得感」でアップセルを促進
```

### 損失回避バイアス

```
FREE → BIZ 課金の設計:

  FREE: 全機能使い放題（AI含む）。ただし保存・エクスポート不可。
  ユーザーが2時間かけて資料を作り込んだ後:
  → 「保存したければ BIZ を購入」

  → 作り込んだ成果物を「失いたくない」のは
    「失う」感覚（損失回避バイアスが発動）
  → 「BIZ版で保存する」は「成果物を守る」感覚
  → Tversky & Kahneman: 損失の痛みは利得の喜びの2倍

  注意: 初回起動時に「FREE では保存できません」を明確に表示。
        2時間作業した後に知るのと、最初に知るのでは印象が全く違う。
```

```
日本市場での端数価格:

  NG: ¥4,999、¥19,999（海外SaaS的すぎて信頼感がない）
  OK: ¥X,980、¥XX,800（日本のSaaS標準的な端数）
  OK: ¥49,800、¥98,000（日本のB2B標準的な端数）

  → 日本では「〜80」「〜800」が自然
  → 「〜99」は海外ECの印象が強く、B2Bでは避ける
```
